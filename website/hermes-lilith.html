Hermes-Lilith

A local-first AI agent that runs entirely on your hardware. No cloud lock-in. No subscription. Control your PC, manage files, browse the web, schedule tasks, and delegate to sub-agents — all through Telegram. 

$ cd Asgard/Hermes-Lilith
$ python install.bat        # Windows global install
$ lilith                    # Run from anywhere
Philosophy

Why local-first?

Hermes-Lilith is built on a simple belief: your data should stay on your machine. The LLM runs through LM Studio (localhost:1234). Memory is stored in local SQLite. Files never leave your drive. Telegram is just the remote control — the brain is at home.

→ No API keys required for the core. LM Studio serves the model locally.
→ Works offline once LM Studio is loaded. Telegram needs internet, but the agent brain doesn't.
→ 48GB RAM + RTX 3060 can run models up to ~27B parameters with CPU offload.
→ Model auto-detection — "auto" in config picks the best loaded model.

The Gateway

The Gateway is the single point of contact for all external interfaces. It exposes REST endpoints for Telegram, file system operations, scheduler tasks, agents, plugins, and memory.

Endpoint Method Purpose
/api/telegram/chat POST Main chat endpoint for Telegram messages
/api/telegram/pregunta_rapida POST Quick questions (no context persistence)
/api/telegram/confirm POST Human-in-the-loop confirmation
/api/pc/fs GET/POST File system operations
/api/scheduler/tasks GET/POST List or create scheduled tasks
/api/agents GET/POST List or spawn sub-agents
/api/plugins GET List available plugins and tools
/api/memory/* Various Vector memory read/write/search
Systems

Core Components

🧠
EnhancedMemory Sentence-transformer embeddings stored in SQLite. Automatic compression of old memories, entity extraction, and context injection into every prompt. The agent remembers who you are and what you did.
🤖
Agent Manager Spawn autonomous sub-agents with isolated contexts for parallel workstreams. Each agent gets its own session, tools, and memory scope. Delegate heavy tasks without blocking the main loop.
📅
Task Scheduler Cron-like scheduling with persistent SQLite storage. Create, list, run, and delete tasks via REST. The scheduler wakes up the agent at the right time to execute background jobs.
🔌
Plugin System Hot-pluggable tools with dynamic discovery. Enable/disable plugins at runtime. Custom tools can be registered by dropping a Python file in the plugins directory. The Gateway auto-discovers new tools.
📖
RAG Pipeline Ingest documents, chunk them, embed with sentence-transformers, and retrieve semantically relevant passages. Build a personal knowledge base that the agent can query in real time.
🖥️
PC Control File system navigation, process management, Windows automation, browser interaction, screenshot capture, and system monitoring. The agent can literally use your computer.

Hardware

Recommended Specs

Hermes-Lilith is designed to run on consumer hardware. You don't need a data center.

Minimum

  • 16GB RAM
  • Any modern CPU
  • ~10GB free disk
  • Models: 7B params (Q4)
  • Slow but functional

Recommended

  • 32GB+ RAM
  • 6+ core CPU
  • NVIDIA GPU with 4GB+ VRAM
  • Models: 13-27B params
  • Fast responses, good context

Reference Build

  • AMD Ryzen 5 5500 (6c/12t)
  • 48GB DDR4
  • NVIDIA RTX 3060 4GB
  • Dual SSD setup
  • Runs 27B models comfortably
