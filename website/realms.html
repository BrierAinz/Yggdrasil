Cosmology

The Nine Realms

Every project in Yggdrasil belongs to exactly one realm. Each realm has a single purpose, strict entry rules, and a defined migration path. This is how order emerges from chaos.

Decision Tree

Where does your idea go?

Follow the tree. No exceptions. The rules keep the ecosystem healthy.

What are you creating? | ├─→ Dashboard / monitoring / scripts → Asgard ├─→ Prototype UI / visual experiment → Alfheim ├─→ App for personal use → Midgard ├─→ Documentation / knowledge base → Svartalfheim ├─→ AI agent / LLM experiment → Vanaheim ├─→ Project > 1 month duration → Jotunheim ├─→ Active development / sprint → Muspelheim ├─→ Assets / configs / templates → Niflheim └─→ Dead project / archive → Helheim
Realms

Realm Directory

Asgard — Core Technology

The golden realm at the crown of Yggdrasil. Houses the core infrastructure: Hermes-Lilith orchestrator, Gateway API, memory systems, scheduler, and plugin architecture. Permanent. Projects here never migrate out.

  • Hermes-Lilith CLI agent
  • FastAPI Gateway (port 8000)
  • EnhancedMemory vector system
  • Task Scheduler
  • Plugin Manager
  • Agent Manager
  • RAG Pipeline

Vanaheim — AI Agents

The realm of magic and artificial intelligence. All AI experiments, bots, LLM fine-tuning, and agent frameworks live here. Projects migrate to Midgard or Svartalfheim when they mature.

  • Telegram Bot (remote control)
  • Lilith Bridge (Gateway client)
  • Agent profiles (Odin, Adan, Eva, Crystal)
  • vanaheim-framework
  • LLM experiments
  • Council system (decisions, voting)

Alfheim — UI Prototypes

The realm of light and visual experimentation. All UI prototypes, dashboards, visual tools, and frontend experiments start here. Migrate to Midgard when production-ready.

  • VSCode Extension for Lilith
  • UI seed / renderer experiments
  • Electron prototypes
  • Dashboard visualizations

Svartalfheim — Knowledge

The dark realm of the dwarves — knowledge, documentation, and craft. All documentation, playbooks, grimorios (architecture decisions), and knowledge bases are forged here. Permanent.

  • Knowledge Base (Lilith docs)
  • Grimorios (architecture & patterns)
  • Playbooks (deploy, onboard, troubleshoot)
  • Technical plans & roadmaps
  • Legacy archives

Muspelheim — Active Dev

The realm of fire — active development and sprints. Maximum 3 projects at once. Everything here must migrate or die within 2 weeks. No project stagnates in Muspelheim.

  • Sprint-mode projects
  • Quick experiments
  • Hotfixes & burnout recovery
  • AutoMode templates
  • Max 3 concurrent projects

Niflheim — Resources

The frozen realm of resources. Datasets, trained models, configuration files, assets, and templates. No code — only raw materials for projects across all realms.

  • Datasets (CIFAR-10, custom)
  • LLM models & checkpoints
  • Configuration templates
  • Resource scripts
  • Asset libraries

Midgard — Personal Apps

The realm of humans — finished personal applications. Apps that are stable, useful, and maintained. The destination for projects that survive Muspelheim and mature.

  • Personal applications
  • Stable tools for daily use
  • Completed prototypes from Alfheim
  • Polished agents from Vanaheim

Jotunheim — Massive Projects

The realm of giants — large-scale projects that exceed 1 month of development. Maximum 2 active giants at once. These are the titans of the ecosystem.

  • Long-term projects (>1 month)
  • Multi-phase architectures
  • Complex integrations
  • Max 2 concurrent giants

Helheim — Graveyard

The realm of the dead. All failed, abandoned, or archived projects end here. Read-only. No development. A museum of lessons learned and code that once lived.

  • Legacy archives
  • Failed experiments
  • Quarantined files (pycache, node_modules)
  • Read-only access
  • Resurrection requires approval
Migration Rules

Project Lifecycle

Projects are living entities. They move between realms based on their maturity, health, and purpose.

From To Condition
Muspelheim Midgard Personal app is stable and ready
Muspelheim Jotunheim Project grows beyond 1 month scope
Muspelheim Helheim Project fails or is abandoned
Alfheim Midgard / Jotunheim Prototype is validated and useful
Vanaheim Svartalfheim Knowledge matures into documentation
Vanaheim Midgard Agent / AI tool is production-ready
Any realm Helheim Project dies or is archived
→ Migration process: last commit with [MIGRATING to X], copy to destination (preserve history), update README, add link in origin.
