# 🔥 Reglas de Muspelheim - Desarrollo Activo

> **Propósito:** Proyectos en desarrollo intenso, código candente, WIP

---

## ✅ Sí Permitido
- Proyectos en sprint activo
- Código inestable cambiando rápido
- Experimentos de corto plazo
- Parches urgentes (Hotfixes/)

## ❌ Prohibido
- Código estable/terminado (migrar a destino)
- Proyectos inactivos >2 semanas
- Más de 3 proyectos simultáneos

---

## Estructura
```
Muspelheim/
├── README.md
├── AutoMode/            # Sistema de ejecución autónoma
│   ├── active/         # Tareas en progreso
│   ├── completed/      # Tareas finalizadas
│   └── failed/         # Tareas fallidas
├── Hotfixes/           # Parches urgentes
├── Burnout/            # Código descartado
└── Docs/               # Guía de Auto-Mode
```

---

## Reglas Específicas

1. **Máximo 3 proyectos**: Si hay 4, uno debe migrar o morir
2. **Todo es temporal**: Se mueve o se muere
3. **Sprint mode**: Cambios rápidos, commits frecuentes
4. **Migración obligatoria**:
   - Completado → Midgard/Jotunheim
   - >2 semanas → Migrar forzosamente
   - Falla → Helheim

## Ciclo de Vida
```
Idea → Muspelheim → [Midgard|Jotunheim|...] (destino)
              ↓
          Helheim (si falla)
```
