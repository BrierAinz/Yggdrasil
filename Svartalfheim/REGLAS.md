# 📚 Reglas de Svartalfheim - Biblioteca Técnica

> **Propósito:** Documentación, playbooks, grimorios de conocimiento

---

## ✅ Sí Permitido
- Documentación técnica (markdown)
- Playbooks y procedimientos
- Grimorios de conocimiento profundo
- Knowledge Base searchable
- Archivos históricos

## ❌ Prohibido
- Código ejecutable (va a otros reinos)
- Binarios
- Datos temporales
- Información desactualizada sin marcar

---

## Estructura
```
Svartalfheim/
├── README.md
├── Knowledge_Base/      # Documentación organizada
│   ├── Lilith_Docs/
│   └── Lilith_Legacy/
├── Playbooks/           # Procedimientos
│   ├── Deployment/
│   ├── Troubleshooting/
│   └── Onboarding/
├── Grimorios/           # Conocimiento profundo
│   ├── Arquitectura/
│   ├── Patrones/
│   └── Decisiones/
└── Scripts/             # Scripts de indexación (no app)
```

---

## Reglas Específicas

1. **Conocimiento persistente**: Lo que aprendes, se documenta
2. **Estructurado y searchable**: Debe poderse consultar
3. **Actualizado o archivado**: Docs obsoletos → Helheim
4. **Lilith indexa esto**: Base para RAG/Archivero
